== Objective

By the end of this guided lab exercise, you will be able to:

* Configure auditd for production security monitoring.
* Interpret raw audit log data.
* Create targeted audit rules for compliance requirements.
* Generate actionable security reports.
* Investigate security events using audit tools.

== Introduction

This hands-on lab teaches you how to configure and use the Linux Audit framework (auditd) for security monitoring and compliance in Red Hat Enterprise Linux.

AuditD is a powerful tool that provides detailed logging of system calls, file access, authentication attempts, and other security events. It is essential for security compliance (STIG, PCI-DSS, HIPAA), forensic investigation, detecting unauthorized access, and monitoring administrative actions.

This lab prepares you for real-world security operations including compliance auditing, forensic investigation, threat detection, and change management.

=== Prerequisites

* Basic Linux command-line experience
* Understanding of file permissions and user management
* Familiarity with system administration concepts

=== Lab Modules

. Installation & Configuration — Set up and configure the audit daemon
. Understanding Audit Logs — Learn to read and interpret audit log entries
. Creating Watch Rules — Monitor specific files with audit rules
. Preconfigured Rules — Deploy STIG-compliant rule sets
. Custom Rule Creation — Write persistent custom audit rules
. Generating Reports — Use aureport and related tools for security analysis

*Lab Note: If Using Red Hat Demo System (RHPDS)*

When you follow the next steps, you will have instructions followed by a small box with text that is meant to be used in the terminal. In the upper right corner there will be a couple of symbols. It should say something like "BASH", an icon of a clipboard, and an icon of a terminal.

You should also be able to click the terminal icon and the command is then auto-typed into the machine terminal.

Alternatively, you should be able to click the clipboard icon in the code box to copy the command, and you will either use the middle mouse button or right click and paste. The keyboard shortcuts for copy and paste (`Ctrl+C` and `Ctrl+V`) won't work in this environment.
