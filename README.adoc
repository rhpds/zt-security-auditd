= AuditD: Security Auditing in RHEL

== Lab Overview

This hands-on lab teaches you how to configure and use the Linux Audit framework (auditd) for security monitoring and compliance in Red Hat Enterprise Linux.

=== What You'll Learn

* Installing and configuring the AuditD daemon
* Understanding audit log structure and record types
* Creating watch rules to monitor files and directories
* Deploying STIG-compliant preconfigured audit rules
* Writing custom audit rules for specific security requirements
* Generating comprehensive security reports with aureport

=== Prerequisites

* Basic Linux command-line experience
* Understanding of file permissions and user management
* Familiarity with system administration concepts

=== Lab Environment

* **Operating System**: Red Hat Enterprise Linux 10
* **Required Packages**: audit, audit-libs (pre-installed)
* **Estimated Time**: 60-90 minutes

=== Lab Modules

1. **Installation & Configuration** - Set up and configure the audit daemon
2. **Understanding Audit Logs** - Learn to read and interpret audit log entries
3. **Creating Watch Rules** - Monitor specific files with audit rules
4. **Preconfigured Rules** - Deploy STIG-compliant rule sets
5. **Custom Rule Creation** - Write persistent custom audit rules
6. **Generating Reports** - Use aureport and related tools for security analysis

=== Skills Gained

Upon completing this lab, you will be able to:

* Configure auditd for production security monitoring
* Interpret raw audit log data
* Create targeted audit rules for compliance requirements
* Generate actionable security reports
* Investigate security events using audit tools

=== Use Cases

This lab prepares you for:

* **Security Compliance** - Meeting STIG, PCI-DSS, HIPAA requirements
* **Forensic Investigation** - Tracking security incidents
* **Threat Detection** - Identifying unauthorized access
* **Change Management** - Auditing administrative actions

== Local Development

To build and preview this lab locally:

[source,bash]
----
cd zt-security-auditd
./utilities/lab-build
./utilities/lab-serve
----

Then open http://localhost:8080 in your browser.

To stop the server:
[source,bash]
----
./utilities/lab-stop
----

To clean build artifacts:
[source,bash]
----
./utilities/lab-clean
----
