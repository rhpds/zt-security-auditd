## Step 4 Creating Watch Rules

Watch rules allow you to monitor specific files or directories for any access or modification. This is crucial for tracking changes to sensitive files like configuration files, password databases, or critical system binaries.


**4.1** Let's create a test file that we'll monitor with audit rules:
[source,bash,role="execute",subs=attributes+]
----
touch /home/rhel/log_everything_and_the_kitchen_sink.txt
----

**4.2** Now let's add some content to this file:
[source,bash,role="execute",subs=attributes+]
----
echo "Logging everything, including the kitchen sink, because why not? Let's see what happens!" > /home/rhel/log_everything_and_the_kitchen_sink.txt
----


**4.3** Now we'll create an audit rule to watch this file. The `auditctl` command allows us to add rules dynamically:
[source,bash,role="execute",subs=attributes+]
----
sudo auditctl -w /home/rhel/log_everything_and_the_kitchen_sink.txt -S all -k rhel-docs-temp
----

Let's break down this command:

* **`-w`** - Specifies the file or directory to watch
* **`-S all`** - Watches all system calls that interact with this file
* **`-k rhel-docs-temp`** - Assigns a custom key for easy filtering in logs

[TIP]
====
The `-k` (key) parameter is extremely useful! It lets you tag audit rules with custom labels, making it easy to search for related events later using `ausearch -k <keyname>`.
====


**4.4** Let's access the file to generate some audit events:
[source,bash,role="execute",subs=attributes+]
----
cat /home/rhel/log_everything_and_the_kitchen_sink.txt
----

**4.5** Now let's use `ausearch` to find all audit events related to our watched file:
[source,bash,role="execute",subs=attributes+]
----
sudo ausearch -f /home/rhel/log_everything_and_the_kitchen_sink.txt
----

You should see audit records showing:
* When the file was created
* When it was written to
* When it was read (from our `cat` command)
* The user who performed each action
* The process that accessed the file

[NOTE]
====
The `-f` flag searches by filename. You can also use `-k rhel-docs-temp` to search by the key we assigned to this rule.
====


**4.6** Let's search using the key we assigned:
[source,bash,role="execute",subs=attributes+]
----
sudo ausearch -k rhel-docs-temp -ts today
----

This is often more convenient than searching by filename, especially when you have multiple files monitored with the same purpose.

== Understanding Watch Rule Persistence

[IMPORTANT]
====
The rule we just created with `auditctl -w` is **temporary**. It will be lost when the system reboots.

In the next modules, we'll learn how to make rules persistent by adding them to `/etc/audit/rules.d/`.
