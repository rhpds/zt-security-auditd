= Installation & Configuration

== Overview

Welcome to the AuditD Security Auditing lab! In this hands-on lab, you'll learn how to configure and use the Linux Audit framework (auditd) to monitor and track security-relevant events on your RHEL system.

AuditD is a powerful tool that provides detailed logging of system calls, file access, authentication attempts, and other security events. It's essential for:

* Security compliance (STIG, PCI-DSS, HIPAA)
* Forensic investigation
* Detecting unauthorized access
* Monitoring administrative actions

[NOTE]
====
This lab uses RHEL 10, where the audit package comes pre-installed. You'll configure it to meet your security requirements.
====

== Step 1: Verify AuditD Installation

First, let's verify that the audit package is installed on your system.

[source,bash,role=execute]
----
dnf list installed audit
----

You should see the audit package listed. The audit daemon is a critical security component in RHEL.

== Step 2: Review Default Configuration

The audit daemon reads its configuration from `/etc/audit/auditd.conf` at boot time. Let's examine the default settings:

[source,bash,role=execute]
----
sudo cat /etc/audit/auditd.conf
----

This configuration file controls various aspects of audit logging, including:

* **log_file** - Where audit logs are stored (default: `/var/log/audit/audit.log`)
* **max_log_file** - Maximum size of a single log file
* **num_logs** - Number of log files to retain
* **space_left** - Threshold for disk space warnings
* **admin_space_left** - Critical disk space threshold

[TIP]
====
The `space_left` parameter is particularly important for production systems. When available disk space reaches this threshold, auditd can trigger alerts or take defensive actions.
====

== Step 3: Modify Configuration Settings

Let's create a backup of the configuration and then modify a setting to demonstrate configuration management.

[source,bash,role=execute]
----
sudo cp /etc/audit/auditd.conf /etc/audit/auditd.conf.backup
----

Now let's increase the `space_left` threshold from 75 MB to 5120 MB (5 GB):

[source,bash,role=execute]
----
sudo sed -i 's/space_left = 75/space_left = 5120/' /etc/audit/auditd.conf
----

== Step 4: Restart AuditD Service

After making configuration changes, we need to restart the audit daemon to apply them:

[source,bash,role=execute]
----
sudo service auditd condrestart
----

Verify the service is running:

[source,bash,role=execute]
----
sudo service auditd status
----

[IMPORTANT]
====
The audit daemon is a special service that doesn't use systemctl for restart operations due to security requirements. Always use the `service` command with auditd.
====

* Verified the AuditD installation
* Reviewed the default configuration in `/etc/audit/auditd.conf`
* Created a configuration backup
* Modified the disk space threshold setting
* Restarted the audit service to apply changes
