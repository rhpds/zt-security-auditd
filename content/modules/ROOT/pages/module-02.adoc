## Step 2: Verify AuditD Installation


**2.1** First, let's verify that the audit package is installed on your system.

[source,bash,role="execute",subs=attributes+]
----
dnf list installed audit
----

You should see the audit package listed. The audit daemon is a critical security component in RHEL.


**2.2** The audit daemon reads its configuration from `/etc/audit/auditd.conf` at boot time. Let's examine the default settings:
[source,bash,role="execute",subs=attributes+]
----
sudo cat /etc/audit/auditd.conf
----

This configuration file controls various aspects of audit logging, including:

* **log_file** - Where audit logs are stored (default: `/var/log/audit/audit.log`)
* **max_log_file** - Maximum size of a single log file
* **num_logs** - Number of log files to retain
* **space_left** - Threshold for disk space warnings
* **admin_space_left** - Critical disk space threshold

[TIP]
====
The `space_left` parameter is particularly important for production systems. When available disk space reaches this threshold, auditd can trigger alerts or take defensive actions.
====

**2.3** Let's create a backup of the configuration and then modify a setting to demonstrate configuration management.
[source,bash,role="execute",subs=attributes+]
----
sudo cp /etc/audit/auditd.conf /etc/audit/auditd.conf.backup
----

**2.4** Now let's increase the `space_left` threshold from 75 MB to 5120 MB (5 GB):
[source,bash,role="execute",subs=attributes+]
----
sudo sed -i 's/space_left = 75/space_left = 5120/' /etc/audit/auditd.conf
----

**2.5** After making configuration changes, we need to restart the audit daemon to apply them:
[source,bash,role="execute",subs=attributes+]
----
sudo service auditd condrestart
----

**2.6** Verify the service is running:
[source,bash,role="execute",subs=attributes+]
----
sudo service auditd status
----

You may need to "Ctrl+c" on your keyboard to close out the systemctl status.


[IMPORTANT]
====
The audit daemon is a special service that doesn't use systemctl for restart operations due to security requirements. Always use the `service` command with auditd.
====
